# 项目介绍

这是一个**基于 Vue.js 开发的电商类单页应用（SPA）**，核心面向移动端/多端适配的电商购物场景，具备完整的购物流程和用户中心功能，以下是分模块拆解的核心功能：

## 一、核心业务模块（views 目录）
这是项目的核心页面载体，覆盖电商全流程：
### 1. 首页（Home/index.vue）
- 电商首页核心功能：轮播图、商品分类入口、推荐商品、秒杀/活动专区等；
- 包含首页布局、数据拉取（如首页推荐商品、分类导航）、页面交互（如下拉刷新、上拉加载）。

### 2. 商品分类（category/Category.vue）
- 商品分类浏览：支持多级分类展示（如左侧分类导航+右侧商品列表）；
- 分类筛选：按分类维度筛选商品，可能包含分类切换、商品排序（价格/销量）等功能。

### 3. 商品列表/商品页（product/Product.vue）
- 商品列表展示：按分类/搜索/筛选条件展示商品列表，支持分页/懒加载；
- 可能包含商品搜索、筛选（价格区间、销量、新品）、排序等功能。

### 4. 商品详情（detail/Detail.vue）
- 商品详情核心：商品图文展示、规格选择（尺寸/颜色）、库存展示、加入购物车/立即购买；
- 附加功能：商品评价、相关推荐、收藏/分享等。

### 5. 购物车（cart/Cart.vue）
- 购物车管理：商品勾选/全选、数量修改、价格计算、删除商品；
- 结算功能：勾选商品跳转下单页面，支持清空购物车。

### 6. 订单模块（order/Order.vue、OrderWait.vue）
- 订单创建：选择收货地址、支付方式、提交订单；
- 订单状态：OrderWait 大概率对应“待付款/待发货/待收货”等状态页，支持取消订单、支付操作、查看物流；
- 历史订单：展示已完成/已取消订单，支持查看详情、再次购买。

### 7. 地址管理（address/AddAddress.vue、Address.vue）
- 地址列表：展示已保存的收货地址，支持默认地址设置、编辑/删除；
- 新增/编辑地址：填写省市区（assets/js/areaList.js 应为地区数据）、详细地址、收货人、电话，表单校验。

### 8. 用户中心（member 目录）
- 登录/注册（Login.vue、Register.vue）：账号密码/手机号验证码登录、注册，表单校验（utils/strategies.js 应为校验策略）；
- 个人信息（Info.vue、Set.vue）：查看/编辑个人资料（头像/昵称）、修改密码、退出登录；
- 会员主页（Member.vue）：聚合个人信息、订单入口、地址入口、收藏/优惠券等。

## 二、技术支撑模块
### 1. 网络请求（utils/axios.js）
- 封装 axios：统一请求拦截（添加 token、设置请求头）、响应拦截（错误处理、token 过期刷新）；
- 接口管理（api 目录）：按业务拆分接口（地址/购物车/分类/订单/商品/上传/用户），统一导出调用。

### 2. 状态管理（store 目录）
- Vuex 核心配置：state（全局状态，如用户信息、购物车数据、分类列表）、mutations（同步修改状态）、actions（异步操作，如拉取购物车数据）、getters（状态计算，如购物车总价）；
- 常量管理（mutation-types.js）：统一定义 mutation 类型，避免硬编码。

### 3. 路由（router.js）
- 路由配置：定义所有页面路由，设置路由守卫（如未登录拦截跳登录页）；
- 路由懒加载：优化首屏加载速度，按模块拆分路由。

### 4. 国际化（lang 目录）
- 多语言支持：en.js（英文）、zh.js（中文）、index.js（初始化 i18n 插件），适配不同语言环境的文本展示。

### 5. 样式与配置
- 样式管理：public/less 下的 base.less（基础样式）、variable.less（样式变量），统一样式规范；
- 工程配置：vue.config.js（Vue 项目自定义配置，如代理、打包优化）、babel.config.js（ES6 转译）、postcss.config.js（CSS 兼容处理）。

### 6. 通用能力
- 混合封装（mixins/dataMixin.js）：提取通用逻辑（如分页数据处理、表单校验），复用在多个组件；
- 组件复用（components/NoPage.vue）：404 页面或空数据占位组件，全局复用；
- 静态资源：public 下的 favicon、图片，src/assets 下的 logo、地区数据等；
- 服务端（server.js）：大概率是本地开发用的 mock 服务/代理服务，或生产环境的静态资源服务器。

## 三、项目整体定位
- **终端适配**：从页面结构和命名来看，核心面向移动端（H5/小程序/webview 嵌入），也兼容 PC 端适配；
- **核心流程**：覆盖“浏览商品→加入购物车→下单支付→收货评价”的电商完整闭环；
- **技术栈**：Vue 3 + Vue Router + Pinia + Axios + Less + Babel + PostCSS，符合主流 Vue 电商项目技术选型；
- **工程化**：按业务模块拆分文件，规范统一，具备可维护性和扩展性。


总结：这是一个功能完整的电商前端项目，覆盖用户从浏览商品到完成订单的全流程，同时具备用户管理、地址管理、订单管理等核心配套功能，技术架构符合 Vue 生态的最佳实践。
